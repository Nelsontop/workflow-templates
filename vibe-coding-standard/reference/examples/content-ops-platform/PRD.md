# Content Ops Platform Example PRD

## 生成约定

- 本示例用于演示“内容矩阵运营后台”类型项目如何填写模板
- 示例强调内容运营主链路：选题、草稿、审核、发布任务、状态回填、基础复盘
- 示例明确预留平台发布接口，但本期不实现真实平台 API 对接
- 未写入本文件的功能，默认不进入当前版本
- 关键用户流程必须能映射到同目录下 `ARCH.md` 的核心工作流与验证方式

## 产品概述

- 产品名称：Content Ops Console
- 一句话定义：一个帮助内容团队管理多账号、多平台内容矩阵的运营后台
- 目标用户：内容运营、审核负责人、内容负责人
- 目标用户当前如何解决这个问题：通过表格、飞书群、平台后台来回切换，内容计划、发布状态和复盘信息难统一

## 背景与问题

- 我们要解决的核心问题是什么：内容矩阵运营流程分散，导致任务断点多、状态不透明、复盘数据难沉淀
- 这个问题为什么现在必须解决：账号数量和内容条目增加后，手工方式已经难以稳定支撑日常发布节奏
- 如果这次不做，代价是什么：内容发布依赖人工追踪，错误率上升，复盘缺少统一口径

### 证据

- 用户研究：运营同学反馈“内容在哪个平台发布了、谁审过、结果怎么样”需要在多个地方查
- 行为数据：暂无统一系统数据，当前以表格和群消息为主
- 支持/销售/运营信号：日常运营复盘依赖人工拼接多个平台后台截图
- 竞品或市场信号：成熟团队通常会用统一运营台管理内容矩阵和发布任务

## 核心用户流程

### 流程 1：创建内容草稿并提交审核

1. 运营进入内容列表页并点击“新建内容”
2. 输入标题、正文、平台计划、发布时间和素材引用
3. 保存草稿
4. 点击“提交审核”使内容进入待审状态

失败分支：

- 哪一步可能失败：字段校验失败、草稿保存失败、状态流转失败
- 失败时如何提示：字段级错误或页面级错误提示
- 用户下一步可以做什么：修正内容后重试保存或再次提交审核

### 流程 2：审核通过并生成发布任务

1. 审核负责人进入待审列表
2. 查看内容详情和平台计划
3. 选择“通过”或“打回”
4. 通过后系统自动生成平台维度发布任务

失败分支：

- 哪一步可能失败：内容数据缺失、审核操作失败、任务生成失败
- 失败时如何提示：显示具体失败原因，避免误以为内容已进入发布
- 用户下一步可以做什么：补全内容、重新操作，或联系技术排查

### 流程 3：触发发布并回填状态

1. 运营在发布任务页选择一条待发布任务
2. 点击“发布”或由本地 CLI / 技能触发统一发布接口
3. 系统把任务交给平台发布适配接口
4. 发布结果回填为成功、失败或待重试

失败分支：

- 哪一步可能失败：触发命令失败、平台适配层返回错误、结果回填失败
- 失败时如何提示：任务状态显示失败原因和最近一次尝试时间
- 用户下一步可以做什么：手动重试、修改内容后重发、标记为人工处理

### 流程 4：查看基础复盘

1. 内容负责人进入数据概览页
2. 查看按平台和账号聚合的发布状态
3. 查看已成功、失败、待处理任务数量
4. 对异常任务做人工跟进

失败分支：

- 哪一步可能失败：数据回填不完整、聚合失败、任务状态过期
- 失败时如何提示：显示“数据待同步”或“部分结果缺失”
- 用户下一步可以做什么：刷新、手动同步或查看任务详情

补充要求：

- 每条核心用户流程都应能映射到 `ARCH.md` 中至少一个核心工作流
- 若流程涉及等待、异步处理、外部服务或人工介入，需在 PRD 先写清用户预期结果
- 若流程失败后允许重试、撤销或回退，需写清用户可执行动作

## 数据与业务规则

| 实体 | 字段 | 业务含义 | 约束/校验 |
| --- | --- | --- | --- |
| content_item | `title` | 内容标题 | 必填，最多 80 字符 |
| content_item | `body` | 内容正文 | 必填 |
| content_item | `status` | 草稿/待审/已通过/打回 | 系统维护的流转状态 |
| publish_task | `channel` | 平台标识 | 必填，枚举值 |

补充规则：

- 哪些字段用户可编辑：标题、正文、平台计划、发布时间、素材引用
- 哪些数据只读：任务状态、最近一次发布时间、平台返回结果

## 发布与回滚

- 计划如何发布：先发布到内部运营测试环境
- 是否灰度：是，先由单个运营小组试用
- 触发回滚的条件是什么：审核流转不可用、任务状态错乱、统一发布接口无法触发、回填结果错误率过高
- 回滚后用户侧预期表现是什么：回到表格 + 手工平台后台操作方式
- 回滚后哪些数据或状态可能受影响：已创建但未导出的任务记录需要人工比对
- 发布后重点观察哪些指标：任务创建成功率、审核操作成功率、发布触发成功率、结果回填成功率
