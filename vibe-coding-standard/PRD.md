# PRD Template

## 生成约定

- 首稿应先锁定用户价值、范围边界、版本取舍和验收口径
- 如果产品高度依赖视觉和交互，需补充界面与交互层面的需求约束
- 最终内容必须沉淀回本文件，不能只停留在聊天记录
- 未写入本文件的功能，默认不进入当前版本

## 交付物技能参考

| 技能 | 适用交付物 | 适用场景 |
| --- | --- | --- |
| `frontend-dev` | Web 页面、营销页、产品页、仪表盘 | 页面需要强视觉表现、动效、媒体资产或转化文案时使用 |
| `gif-sticker-maker` | GIF 贴纸、表情包、头像动画 | 需要把人物、宠物、物体或 logo 转成 4 张带文案的动态贴纸时使用 |
| `minimax-docx` | Word / `.docx` 正式文档 | 需要产出、编辑、套模板或排版报告、方案、合同、公文等正式文档时使用 |
| `minimax-pdf` | 高视觉质量 PDF | 需要生成、补全表单或重排设计感较强、可打印的 PDF 时使用 |
| `minimax-xlsx` | Excel / `.xlsx` / `.csv` / `.tsv` | 需要创建、读取、分析、修改、校验电子表格、财务模型或透视表时使用 |
| `pptx-generator` | PowerPoint / `.pptx` 演示文稿 | 需要创建、编辑、读取幻灯片、汇报材料或演示 deck 时使用 |
