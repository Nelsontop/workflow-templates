# Security

这里记录模板默认从严的安全边界。

## 默认要求

- 密钥只放服务端环境变量，不写入前端
- `.env` 不入库，使用 `.env.example` 记录变量名
- 服务端接口必须做鉴权，不信任前端参数
- 涉及用户数据的查询必须按当前用户身份过滤
- 登录、验证码、重置密码等敏感接口必须有频率限制

## 以下主题必须先写文档，再实现

- 认证方案
- 支付流程
- 数据留存与删除
- 第三方依赖失败后的降级策略
- 关键系统间 handoff 契约
- 关键状态的失败恢复方式

## 评审问题

- 当前变更是否引入新的敏感数据流？
- 是否有仅靠前端约束的关键校验？
- 是否出现无法追踪或无法撤销的危险操作？
