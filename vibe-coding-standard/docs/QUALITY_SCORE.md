# Quality Score

本文件定义模板鼓励的质量评估维度。

## 评分维度

| 维度 | 关注点 | 最低要求 |
| --- | --- | --- |
| 文档完整性 | 是否能找到正确入口、计划、验收与边界 | 当前切片必须可定位到计划和状态 |
| 架构清晰度 | 模块职责、边界和目录是否可推理 | 不把 UI、状态、数据访问和业务规则混在一起 |
| 验证闭环 | 是否有 lint、类型检查、测试或手动回归记录 | 缺失项必须显式记录 |
| 可维护性 | 命名、分层、文件规模、重复代码 | 出现重复模式时优先沉淀模板或工具 |
| 反熵能力 | 是否能发现和清理漂移 | 技术债进入跟踪文件，不留在聊天里 |

## 使用方式

- 可以按切片做定性自评
- 可以为未来 lint 或 CI 规则提供检查维度
- 当一个问题反复出现时，应把它从“评语”升级成“可执行约束”
