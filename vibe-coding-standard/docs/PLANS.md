# Plans

执行计划是本模板中的一级工件。

## 目录

- `exec-plans/active/`
  当前正在推进的切片计划
- `exec-plans/completed/`
  已完成且可追溯的计划
- `exec-plans/tech-debt-tracker.md`
  已知技术债、反熵任务和计划外修补项

## 什么时候必须建计划

满足任一条件就应建计划：

- 任务跨多个文件或模块
- 需要多步验证
- 需要与 PRD、设计或安全文档对齐
- 需要后续 agent 接手
- 你已经开始在脑中分步骤做事

## 计划模板

每个计划至少包含：

- 目标
- 当前状态：`draft` / `active` / `blocked` / `done`
- 本次明确不做
- 允许修改的文件或目录
- 依赖文档
- 实施步骤
- 验收标准
- 验证记录
- 遗留问题

## 流转规则

1. 新计划先放入 `active/`
2. 实施中持续更新状态和验证记录
3. 完成后移入 `completed/`
4. 若中途中止，标记 `blocked` 并写清阻塞原因
5. 计划外发现的长期问题进入 `tech-debt-tracker.md`

## 与 project_state 的关系

- `project_state.md` 只保留“当前激活的是哪一个计划”以及简短摘要
- 详细步骤、判断过程和验证日志不要塞回 `project_state.md`

## 风格要求

- 一个计划只描述一个可验收切片
- 步骤按顺序可执行
- 范围边界必须明确
- 不用“顺手优化”“必要时重构”这类模糊描述
