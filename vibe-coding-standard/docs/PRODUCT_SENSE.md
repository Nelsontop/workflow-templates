# Product Sense

这里记录产品判断的默认原则，用于补足 PRD 之外的取舍方式。

## 默认原则

- 先验证真实用户价值，再扩能力
- 非目标要和目标一样明确
- 不要把“可以做”误当成“现在就该做”
- 如果当前版本只是验证阶段，应优先缩短从入口到结果的路径
- 如果需求边界不稳，不要提前展开大规模后端或基础设施改造

## 需求取舍问题

遇到以下情况时，先停下来判断再实现：

- 用户口头要求与现有规格冲突
- 新需求会打破当前切片边界
- 技术上可行，但无法在本期清楚验收
- 需要额外引入长期维护成本

## 版本控制

- 当前版本只做能独立验收的最小闭环
- 未来版本的设想可以记，但不要混入本期承诺
