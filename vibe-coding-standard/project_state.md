## 最近一次验证

- 手动验证：`做了什么人工检查，结果如何`
- lint：`已运行 / 未运行 / 不适用`
- typecheck：`已运行 / 未运行 / 不适用`
- test：`已运行 / 未运行 / 不适用`
- regression：`是否做了回归检查，覆盖了什么`
- preflight：`如有发布前检查或环境检查，写在这里；否则写 不适用`
- commit：`最近一次与当前切片相关的 commit；没有则写 未提交`

## 维护规则

- 这里只保留当前激活切片的摘要，不写完整实施步骤。
- 详细步骤、判断过程和验证日志写入 `docs/exec-plans/`。
- 切片切换后，及时更新“当前目标”“当前切片”“当前计划”和“最近一次验证”。
- 如果没有对应计划文件，应先补计划，再继续推进中等及以上复杂度任务。
